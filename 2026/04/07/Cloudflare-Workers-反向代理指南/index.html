Cloudflare Workers 反向代理指南

发布时间：2026-01-17

前期准备

必需条件

  1. Cloudflare 账号
  2. 托管在 Cloudflare 上面的域名

注意事项

  • 同一个反代建议不要给很多人用，否则流量过大可能违反 CF 的政策导致被封号
  • 建议小范围自用

没有域名怎么办？

可以白嫖一个免费域名，参考这篇教程：https://zhuanlan.zhihu.com/p/1932505765198926962

1. 创建 Worker

  1. 打开 Cloudflare 面板
  2. 在左侧菜单找到 Workers 和 Pages
  3. 点击 创建应用程序
  4. 模板选择 Hello World
  5. 开始新建一个 Worker

3. 部署

点击右上角的 部署 (Deploy) 按钮保存。

可选：如果你会混淆加密 JS 代码，可以自行处理一下防止被 CF 特征识别。

第二步：域名 CNAME 优选

1. 添加 DNS 记录

  1. 打开你托管在 Cloudflare 的域名管理页面
  2. 进入 DNS 记录
  3. 添加一条 CNAME 记录：
字段 填写内容
名称 填写你想要的前缀（例如 emby）
目标 填写一个优选域名
代理状态 务必关闭（灰色云朵）

2. 获取优选域名

优选域名可以在这里选择：https://cf.090227.xyz/

第三步：配置路由

这是最关键的一步，让你的域名流量经过刚才创建的 Worker。

1. 进入路由设置

在你的域名管理页面左侧菜单栏，点击 Workers 路由 (Workers Routes)。

2. 添加路由

  1. 点击 添加路由
  2. 路由一栏输入：示例：emby.你的域名.com/*

    注意！！！一定要带上 *

  3. Worker 一栏选择你第一步创建的那个 Worker
  4. 点击 保存

完成测试

现在打开你在 DNS 设置的那个域名（例如 https://emby.你的域名.com），应该就可以直接访问你的 Emby 了，并且走的是优选线路。

重要补充

1. DNS 解析要求

  • 反代需要使用国内 DNS 解析，否则可能连不上
  • 如果你有同时使用代理软件，请自行添加规则，把你自己的域名走直连
  • DNS 要用国内的

2. 播放器配置

在播放器上，服务器端口可以不填（默认 443），也可以填以下端口：

  • 443
  • 2053
  • 2083
  • 2087
  • 2096
  • 8443

使用其它端口也许可以在晚高峰提升速度（不确定）。

3. Hills 用户

本方式搭建的反代，使用 Hills 的可以直接在它的服务器线路中添加。

使用说明

  1. 修改 paths 中的内容：左边是路径前缀，右边是对应的服务器地址
  2. 修改 main 中的地址：这是直接访问域名（不带前缀）时连接的默认服务器
